#project summary

    1.本项目实现的东西分别有: 

        首页展现  登录注册  列表渲染  商品分类  购物车增删改算  我的页面  订单详情  添加地址  所用到的数据均是拿线上的数据  数据选自7717食品安全百度_商城  用到标题的地方拿mock数据进行渲染

    2.本项目封装的组件有: 

        弹出框  轮播图  商品列表  购物车模块  邮寄地址  筛选模块  http请求

    3.本项目技术的选型有: 

        1.React的灵活操作 

        2.redux多交互多数据源的特点  
        
        3.react-router路由(react-router-dom和react-router-native)提供核心的路由组件与函数
        
        4.react-redux(其实react-redux就是为了提供方便的容器组件,在react-redux中的connect起到了至关重要的作用它里面的四个参数都被广泛的应用到其中前两个参数[mapStateToProps,mapDispatchToProps]应用比较多后面还有两个参数就是[mergeProps,options]都是应用不多)
        
        5.redux-sage(解决异步的sage是用来分发action进行处理，使体系流程更加清晰)它是一个接收事件,并可能触发新事件的过程管理者，为你的应用管理复杂的流程

        6.移动端和pc端的适应  
        
        7.fetch的封装(为了更方便的提供接口) 
        
        8.node(利用node搭建静态服务进行模拟为了更快速的适应公司的后台)  
        
        9.webpack(自行搭建脚手架理解脚手架的原理以及路由)

    4.项目的整体流程:

        首页 => { // home
            商品列表的渲染运用的接口请求线上的数据然后利用goodsCom里面封装的组件就行渲染加入了上拉加载功能,通过判断scroll的值以及浏览器的高度来决定是否加载
            首页的header是简单的排版
            轮播图是利用swiper进行组件封装
            图标是运用state写了简单的一些title然后用map进行遍历
        }

        搜索页 => { // search
            利用在首页的header中input框的点击跳转跳到search页(附search页只有简单的一些React应有的东西后期做的时候再去添加)
        }

        详情页 => { // detail
            页面首页的数据出来之后点击跳转详情页(未做)
        }

        (注册,登录)页 => { // register  login
            里面用到的东西是接口的开发注册夜和登录页的接口然后在每个固定的页面上写出相应的代码在server.js中用到了post请求将每次注册的数据存在预先准备好的user.json中,然后每次进行比较是否有之前注册过的;利用user和password进行相应的后台数据对比在弹出,通过判断value值来进行对比
        }

        分类页 => { // classify
            在分类页中应用的也是同样的逻辑在server.js中注册号接口然后在classify中只写出了左边的列表右边的列表后期添上而且需要点击左边的单个盒子实现出右边的数据渲染
        }

        购物车页 => { // cart
            购物车的逻辑相对来说用到的东西比较混杂首先购物车的页面开发需要用到前面的列表数据只有在确定是否添加到了购物车之后才能进行购物车的开发在goodsCom中有购物车的逻辑运维,同时购物车的"编辑"加上了onclick通过点击来判断是否删除选中的商品信息通过(id)的遍历判断选中的事哪一个,添加购物车的思想取决于首页的商品信息,通过首页里面的购物车标识点击之后弹出封装好的提示框(封装好的组件)直接添加到购物车中,购物车中的计算通过"price"and"count"来计算,所有的添加、删除都是封装的组件通过调用实现效果的实现
        }

        我的页 => { // mine
            我的页面通过之前的注册登录页面通过cookie来进行后面数据的存储,点击退出时会直接跳回首页然而再次点击我的路由的时候不会出现之前的登录信息,将会弹出登录页然后重新登录自己的信息,在我的页面中会有添加地址以及我的订单和账户余额,再添加地址的时候用到的事三级联动,首先去网上下载一些数据以便后期使用,然后一次便利里面的属性,通过自己的选择添加地址,然后可以删除添加好的地址(逻辑未清晰,正在努力补习中)
        }

    5.项目总结；

        项目的开始是先手搭脚手架运用的都是一些基础的东西webpack和webpack-dev-server 服务是node的静态服务运用脚手架起来服务之后就行路由的搭建运用的有react-router以及里面的部分组件之后封装所有的页面每一个页面为一个文件夹里面的逻辑在后面的每一页中均有体现做到点击路由实现各个路由的跳转用footitem将所有的页面中的路由组合起来运用的逻辑是router以及各种router的组件在路由中应用的authorization为管理权限,通过判断权限来实现登录和首页的切换后期会有项目的压缩打包上线,然后可以再api.js中设置eje模板引擎,在整体的项目开发流程中关键点在于webpack的灵活使用,React的熟练程度,React-router路由的各种操作以及比较混杂的购物车添加逻辑,整体项目的优势在于管理权限的使用,可以让程序更完善的运行,调用的东西很多,swiper的使用也是一项特色,cookie的应用面面俱到,redux以及redux-sage的操作,mock数据的应用,react生命周期函数的需求,组件封装的基础,所有的操作均是react有关的总结！(做的项目有些功能没有实现等后期会加班补上)
